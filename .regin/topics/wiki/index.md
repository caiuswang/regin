## Topics

- [Pattern Conceal Experiments](pattern-experiments.md) - Understand the experiment system that conceals selected H2/H3 sections of a pattern's SKILL.md at deploy time so rule-trigger rates can be compared against an unconcealed baseline. Covers the pure conceal filter, the experiments table and RuleTrigger.experiment_id linkage, the one-active-per-pattern lifecycle, the redeploy hook into skill_sync/skill_deployer, the baseline-vs-variant rollup endpoints, the Vue UI surfaces, and the gap between the ingest path and the rollup's variant column.
- [Proposal review comments (feedback threads anchored to topics & wiki)](proposal-review-comments.md) - Document the inline review-comment system layered on top of a proposal run: how a thread is anchored (general / topic field / proposal summary / wiki range), what HTTP endpoints CRUD threads and comments, how the ORM stores them with revision pointers, how `format_review_feedback_for_prompt` injects open threads into the next regenerate, and how the auto-`addressed` sweep closes threads whose anchored content actually changed.
- [Rule engine design (adapter protocol, registry, PostToolUse driver, add-an-engine playbook)](rule-engine-design.md) - Explain how regin loads, dispatches, and executes lint/rewrite rules: the engine-agnostic Protocol, the three-source registry precedence (explicit settings → legacy grit fallback → autodiscovered bundles), the three concrete adapters (Grit, Bundle, Radon), the PostToolUse hook that drives them per edit, and the step-by-step playbook for adding a new engine. Use this when adding a new engine kind, packaging a new rule pack as a bundle, wiring an existing engine into a new language, or debugging why a rule fired (or didn't) on a file.
- [Session trace design (OTel-style spans, ingest pipeline, aggregates, dashboard read paths)](session-trace-design.md) - Explain how Claude Code sessions are captured as OpenTelemetry-style spans by hook handlers, persisted through the trace_service ingest layer into session_spans plus a denormalized sessions row, and rendered by the Vue SessionTraceView via the /api/sessions/* read endpoints.
- [Topic proposal pipeline (request → draft → review → apply/stop)](topic-proposal-pipeline.md) - Explain the end-to-end funnel that turns a user's topic request into one row in `.regin/topics/topic.json`: an external tool-using agent drafts the JSON+wiki, regin validates and persists it via the ORM, reviewers iterate through revise/regenerate/restore/stop, and a single server-side /diff+/apply path (or the matching library accept/replace/merge helpers) promotes a topic into the approved graph. Also describes the cross-thread cancel machinery that lets a Stop request terminate the agent subprocess without racing the worker's terminal-state writes.
- [Topic routing (approved graph + candidate fallback)](topic-routing.md) - Understand how an agent resolves a freeform query into approved topic context: alias / label / ref-path / multi-keyword scoring against `.regin/topics/topic.json`, with optional fallback to unapproved candidates in `.regin/topics/candidates.json`. Covers the matcher (`match_topic`), the routing envelope (`route_topic`) that returns ordered refs + bounded wiki pages + first-degree related topics, and the deployed `topic-router` skill that tells agents how to use it.
