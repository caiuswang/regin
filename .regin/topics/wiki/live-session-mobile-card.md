# /live mobile session-tail card

`/live/:id?` (`frontend/src/router.js`, route name `live`) is a mobile-first, single-card view of one session's spans — the phone-sized alternative to the desktop `SessionTraceView`. Design intent lives in `docs/mobile-progress-card-design.md`: a terminal-style recent-spans card with fold-earlier, a live tail, and minimal chrome, explicitly **not** the `send_to_user` message digest that `InboxView` renders. The card also doubles as a remote control: when the session's terminal is bridge-reachable it can steer the running agent and answer its questions (`docs/agent-bridge-design.md`).

## Shape: one card

`LiveSessionView.vue` renders exactly one `.live-card`:

1. **Header** (`.live-hd`) — a status dot + label driven by the mirrored NOW-zone state: `live-status-ended` (`✓ finished`), `live-status-idle` (steady dot, `idle`), or `live-status-running` (pulsing, `running`). Beside it: an elapsed/last-seen line (`elapsedLabel`), the session goal (`live-goal`, tap to expand), and two icon buttons — switch session (`live-switch`) and filter/search (`live-filter`, which shows a `live-hd-badge` dot when any filter is active). The header gains `live-hd-scrolled` once the tail scrolls past 4px.
2. **Fold row** (`.live-fold`) — the first child of the tail, shown only when `hasMoreOlder`; it displays `earlierCount` not-yet-loaded earlier spans and loads one older turn page on tap, anchoring the scroll position across the prepend (`unfold()` records `scrollHeight`/`scrollTop` before `loadOlder()` and restores the offset after).
3. **Tail** (`.live-tail`, `ref=tailEl`) — the *only* scroll region. Renders a `LiveTailRow` for every row that survives `filterSpans`, plus empty states (`no spans yet`, `no spans match the current filter`, loading/error). A follow-tail rule keeps the view pinned while new spans arrive (`isPinned()` — within 40px of the bottom); when the user has scrolled up, arrivals increment a `↓ N new` chip (`.live-newchip`) that scrolls back down on tap. A blinking caret (`caretSpanId`) rides the newest visible row while the session runs, suppressed when idle or ended.
4. **NOW zone** (`LiveNowZone.vue`, `.live-now`) — a sticky footer, at most one state at a time (see below).

Row-detail, full-message, filter, Q&A, and session-switcher interactions all happen in **bottom sheets** (`LiveSheet.vue`), positioned absolutely *inside* the card (not a portal) so the tail's scroll survives open/close — `openSheet` saves `tailEl.scrollTop` and `closeSheet` restores it. `LiveSheet` owns the copy-to-clipboard affordance (`live-sheet-copy`) shared by every sheet header. The single `sheet` ref carries a `{ kind, spanId }` where kind is `message` | `detail` | `qa` | `sessions` | `filter`.

## Data + poll lifecycle: `useLiveTail`

`frontend/src/composables/useLiveTail.js` is the sole data/lifecycle owner; the view is presentation-only. It rides the same shallow-map pagination the desktop trace view's `useTraceData` uses:

- **Window unit is turn anchors, not spans.** `PAGE_SIZE = 5` pages `/api/sessions/<id>/map?shallow=1&limit=5` by turn-anchor count; activity spans (tool calls, responses, PENDING placeholders) are children of those anchors and stream in separately via `/api/sessions/<id>/spans/<root>/children?deep=1`, fetched with bounded concurrency (`CHILD_CONCURRENCY = 4`). Both endpoints live in `web/blueprints/trace/sessions.py` (`api_session_map`, `api_session_span_children`), shared with the desktop view.
- **Poll cadence**: every `POLL_ACTIVE_MS` (4s) while the session looks active, `POLL_STALE_MS` (15s) otherwise (`active` via the shared `isActiveSession` rule); polling pauses while the tab is hidden and resumes on `visibilitychange`. An `ended` session schedules nothing further.
- **`ended` trusts `status` over `ended_at`.** A resumed session (exit tmux → `claude --resume`) keeps a stale `ended_at`; the computed `ended` returns false when `status==='active'` or when `last_seen` runs more than 1s past `ended_at`, so the card doesn't wrongly read `✓ finished` mid-run.
- **Every poll re-fetches the full newest window** (a `limit` cursor, not `after_id`) because `retired_span_ids` — the ids the serve-time merge dropped from the window (a `promptlive-` prompt promoted to its real anchor, a `pending-`/`permreq-` placeholder resolved) — is only computed from that request's raw rows. An `after_id` poll would return an empty window/retired list whenever no new prompt landed, silently leaking a resolved-but-unpruned placeholder into the card. `pruneStalePending()` is a second safety net: on a poll where every subtree fetch succeeded, any PENDING placeholder absent from that tick's responses is dropped, so a `permreq-`/`pending-` row whose anchor aged out of the 5-anchor window can't pin the NOW zone forever. `refreshSubtrees` deep-refreshes the trailing `DEEP_REFRESH_TAIL = 3` roots plus any earlier `failedRootIds` (retried so a transient 500 doesn't lose a turn).
- **Generation guard (`epoch`)**: `start()` bumps a counter; any async work captured under a stale generation discards its results (`isStale(gen)`), so switching sessions mid-flight can never let a late response from the *old* session merge into the new one.
- **`mergeSpans`** is a keyed, append/update-only merge (pruning is the separate retired-ids path) that prefers non-empty incoming `attributes` over a cached empty shallow placeholder — mirrors `useTraceData.mergeLoadedSpans`. It is exposed so the view can splice lazily-fetched attributes back in (`fetchContent` → `/sessions/<id>/spans/<span>/content`).
- **`appendedSpans`** exposes spans that sort strictly after the previously-last row (a start_time compare, not an id watermark — a rescan-backfilled row can carry a high DB id yet sort mid-list) so the view drives follow-tail / the chip without misreading a filter change as new content.
- **`applySummary`** also carries `bridge_reachable`, `bridge_pane`, and `server_now` off the same poll — no separate bridge polling loop. `server_now` (the server's wall-clock at read time, in `web/blueprints/trace/sessions.py`) plus a phone-clock stamp (`server_now_at = Date.now()`) let the NOW zone's elapsed anchor to the server clock so a viewer in a different timezone doesn't leak the offset.

## Row semantics: `utils/liveRows.js`

Pure, stateless functions, importing label/color/category primitives from `traceFormatters.js` rather than duplicating them. What's specific to the live card:

- **`rowKind`** splits every span into `msg` (prompt / assistant_response — an eyebrow + up-to-4-line body), `qa` (ask-user-question / permission — its own delicate 2-line mini card), or `act` (everything else — a faint one-line micro-row whose dot color carries its category).
- **`isQaSpan` / `qaRowModel`** back the `qa` row: `tool.AskUserQuestion`, `permission.request`, or any `permreq-` id. `askRowModel` renders the question glyph `?`, an eyebrow (`Ask user · +N more`), the question text, and an outcome line (`✓ <chosen>` / `✗ denied` / `… waiting for your answer`); `permRowModel` renders `⚠ Permission · <tool>`, the `$ <command_preview>`, and granted/denied/waiting.
- **`isSignal`** is the default show/hide tier: an allow-list (`prompt`, `assistant_response`, `subagent.start/stop`, `file.edit`, `plan.edit`, `memory.recall`, `permission.request`), plus `tool.*`/`skill.*` prefixes (but never `pre_tool.*`), plus any `rule.check` with findings — errors/denials always show. Anything else (including any *new*, unrecognized span type) is bucketed as system and hidden unless the user flips "Show system spans" — a default-closed policy so novel span types never flood the card.
- **`isHotSpan`** keeps a full-saturation dot for edits / failures / denials (`EDIT_TOOL_NAMES` + shared error/denied helpers); every other row renders its category color as muted texture.
- **`humanMain`** renders "what happened", not the raw span type, via a keyed `MAIN_BUILDERS` dispatch + `humanToolMain` (a `tool.Read` on `foo.py` → `Read foo.py`, a `tool.Bash` → a `$ <command_preview>` mono line). Every signal-tier span name must have a phrasing here — raw span-type names must never reach the default view (acceptance 8c). `stripMarkdown` (shared with the NOW zone's clamp) flattens markdown while preserving snake_case identifiers.
- **`filterSpans`** is the single visible-row predicate combining the signal tier, the active category chip, and the search query (same haystack the desktop Terminal searches, via `spanMatchesSearch`). Chip counts are computed over the *same* tier-filtered list the rows use (`countByCategory(tierSpans)`) so a chip's count always equals its selectable rows.

## NOW zone: `LiveNowZone.vue`

A pure projection of the already-loaded tail, one state at a time by priority: **permission** (`permreq-*` PENDING) › **question** (`pending-*` PENDING AskUserQuestion) › **tool** (other `pending-*` PENDING) › **prompt** (`promptlive-*` PENDING) › **finished** (session ended) › **idle** (alive + bridge-reachable) › **response** (latest `assistant_response`, 2-line clamp, `more ▾`). Permission and question get the amber `live-now-attention` treatment; question surfaces the question text + option count with an `options ▾` that opens the Q&A sheet.

- **Idle is debounced.** `idle` is inferred from the *absence* of any pending span, but a turn also has no pending span mid-flight (between a tool's PostToolUse and the next PreToolUse, or while the agent thinks). `IDLE_SETTLE_MS = 6000` requires the idle condition to hold before believing the agent is actually waiting; until then the working `response` view shows. The resolved `state` is emitted via `state-change` so the header dot/label and caret track it.
- **Timezone-safe elapsed.** For a PENDING permission/question/tool the elapsed anchors to `(server_now − span.start_time) + (phone_now − server_now_at)` — both terms server-local, so a phone in another timezone doesn't leak the offset (a tool would otherwise read `4h00m`). The 1s tick runs only while a tool/permission/question state shows it. `STALE_PENDING_MS = 45000` suppresses the clock when fresher activity exists beyond the pending span (timing a subagent parent / orphaned placeholder, not the current step). `fmtElapsedSeconds` rolls seconds over to `8m09s` / `1h05m`.

## Bridge composer + answerable Q&A

When `bridge_reachable` is true the NOW zone mounts `LiveComposer.vue` (a child, so all async send state stays out of the projection): a full-width composer in `idle` (starts the next turn) and a compact steer variant in `response`/`tool`/`prompt` (queues into the running turn); absent in question/permission/finished or when unreachable. It POSTs the typed text to the web-JWT proxy `/api/sessions/<id>/bridge-send` (`web/blueprints/bridge.py`), which delivers it into the session's registered tmux pane. Delivery is confirmed only when the poll returns the real `promptlive-`/`prompt` span — no client-stamped row is ever appended. An `esc` button POSTs `/bridge-key` to dismiss a stuck terminal overlay. The draft is a `v-model:draft` owned by `LiveNowZone` (which never unmounts), so text typed mid-draft survives a one-poll reachability blip that unmounts the composer, and resets only on a session switch. `LiveCommandMenu.vue` + `useSlashCommands.js` provide `/command`/skill autocomplete over `/api/sessions/<id>/bridge-commands`.

`LiveQaSheet.vue` is the full Q&A detail behind a qa row and the NOW zone's `options ▾`. For a PENDING single-question ask over a reachable bridge it is **answerable**: each option is a button that POSTs `/bridge-answer` (driving the pane's select TUI), plus a free-text input for the "Type something" entry. Multi-question asks and unreachable/completed spans stay read-only — blindly walking a multi-question TUI could answer the wrong sub-question.

## Session switcher: `LiveSessionPicker.vue`

Opened from the header's list icon into a `sessions` bottom sheet. Fetches `GET /sessions?kind=real&size=20` per mount, sorts active-first then newest-`last_seen`-first, and emits `select`; `onPickSession` then `router.push('/live/' + trace_id)`. `kind=real` excludes Playwright/test fixture sessions from the list, but a direct `/live/<test-session-id>` deep link still resolves because `useLiveTail.resolveSessionRow` looks up `kind=all`. Active/inactive sort and the header dot both use `utils/sessionActivity.js`'s `isActiveSession` (the same rule as the desktop Sessions table's green badge): `status==='active'` → active, `status==='ended'` → not, else a 10-minute `last_seen` recency window. `parseLocalIso` in the same file parses the backend's naive local-time ISO strings that native `Date()` would mis-read as UTC.

## Testing

`frontend/tests/live-card.spec.js` is organized by acceptance criterion: the 375px invariant, fold-row states, poll lifecycle, filter sheet, message-first hierarchy, bottom sheets, incremental tail arrival, the NOW-zone state priority (including the dedicated question state and the qa rows), the timezone-safe elapsed rollover, and the idle state + bridge composer / send lifecycle (bridge scenarios patch the map response in-flight and stub the `/bridge-send`/`/bridge-answer` proxies, since the test server runs with the bridge disabled). `frontend/tests/live-picker.spec.js` covers the session switcher; `frontend/tests/responsive.spec.js` covers the broader mobile/responsive layout pass.

## Relationship to the desktop trace view

The live card is a sibling read path to `SessionTraceView`'s **conversation** mode, not a replacement — see **session-trace-design** and **trace-merge-reconcile** for the shared shallow-map/serve-time-merge model both views build on, and **webui-surface** for where `/live` sits in the SPA route/nav map.
